LIBRARY BACKUP – HELP AND INFORMATION
=====================================

Version date: 2026-07-12


PART 1 – USER GUIDE
===================

1. Purpose
----------

Library Backup copies personal Windows folders to a USB flash drive, external
disk, or another selected drive. It can also restore those files safely.

Supported folders:

  - Desktop
  - Documents
  - Downloads
  - Pictures
  - Music
  - Videos
  - Favorites
  - Saved Games
  - Contacts

AppData, temporary files, and cache folders are intentionally excluded.


2. Starting the application
---------------------------

Normally, double-click "Sicherung.lnk". The VBS and BAT launchers can also be
used. The application automatically uses German on a German Windows display
language and English on all other display languages.


3. Creating a backup
--------------------

  1. Select "Back up" at the top.
  2. Connect a USB drive or external disk.
  3. Select the destination drive.
  4. Select the folders to include.
  5. Click "Start backup".
  6. Wait for the successful completion message.

The destination drive must remain connected while the operation is running.


4. Backup behavior
------------------

  - New and changed files are copied.
  - Newer destination files are not replaced by older source files.
  - Files are never automatically deleted from the backup.
  - Robocopy /MIR and /PURGE are not used.
  - Locked files may be skipped and reported in the log.
  - Nothing is written to the destination drive before confirmation.

This is a continuously updated safety copy, not a versioned archive with
multiple historical snapshots.


5. Disk space and FAT32
-----------------------

The preflight check estimates the required additional disk space. The
operation is stopped if the destination does not have enough free space.

If the preflight check cannot read some files or folders (for example
because they are locked), a warning with examples is shown. The backup
continues only after explicit confirmation, because the disk-space estimate
may then be too low. During a restore, read errors on the backup drive still
stop the operation, as they can indicate a failing drive.

FAT32 cannot store a single file of 4 GB or larger. NTFS or exFAT is
recommended for backup drives. Formatting a drive deletes its existing data;
never format a drive without checking its contents first.


6. Restoring files
------------------

  1. Connect the drive containing the backup.
  2. Select "Restore".
  3. Select the backup drive and folders.
  4. Click "Review restore".
  5. Review the conflict preview carefully.
  6. Confirm only if the displayed changes are correct.

The preview shows missing local files, local files that would be replaced,
newer local files that remain protected, the planned data volume, and up to
ten example paths.


7. Restore protection
---------------------

  - Newer local files remain protected by Robocopy /XO.
  - Local files are never deleted.
  - /MIR and /PURGE are not used.
  - Backup metadata must match the current computer and user.
  - Free space is checked separately for every affected local drive.

An older local file can be replaced by a newer copy from the backup. This is
why the conflict preview is mandatory.


8. Cancelling an operation
--------------------------

Use "Cancel backup" or "Cancel restore". The currently processed folder is
allowed to finish safely before the worker exits. Files already copied remain
in place.


9. Backup location
------------------

Backups use this structure:

  <Drive>:\Bibliothekssicherung\<Computer>_<User>\

The folder contains the backed-up folders, _Sicherungsinfo.txt, and the _logs
directory. Internal folder names remain stable and are not changed when the
display language changes.


10. Logs
--------

Backup logs use robocopy_YYYYMMDD_HHMMSS.log. Restore logs use
restore_YYYYMMDD_HHMMSS.log. They are Unicode files. Robocopy exit codes 0–7
mean success or success with notes; codes 8 and above indicate copy errors.


11. Recommendations
-------------------

  - Back up regularly.
  - Safely eject the drive after completion.
  - Do not leave the backup drive permanently connected.
  - Keep another copy of irreplaceable files.
  - Occasionally test restoring a non-critical file.
  - Review logs that contain errors.


PART 2 – IMPLEMENTATION AND TECH STACK
======================================

12. Components
--------------

Bibliothekssicherung-GUI.ps1 provides the Windows Forms interface. It manages
the language, mode, drive and folder selection, status, conflict preview,
approval, and cancellation.

Bibliothekssicherung.ps1 is the worker for backup and restore. It performs
validation and preflight checks and runs Robocopy.


13. Tech stack
--------------

  - Windows PowerShell 5.1
  - .NET Framework
  - System.Windows.Forms and System.Drawing
  - Robocopy
  - Win32_LogicalDisk via CIM/WMI
  - Windows User Shell Folders registry values
  - JSON for previews and structured results
  - Atomic temporary files for inter-process communication

The scripts use UTF-8 with BOM for Windows PowerShell 5.1 compatibility.


14. Process architecture
------------------------

The responsive GUI starts a separate worker process. The GUI polls an atomic
status file, displays the restore preview or preflight warnings, writes
approval or cancellation signals, and reads the final JSON result.

The worker validates the drive and folder mappings, calculates required disk
space, waits for restore or preflight-warning approval, runs Robocopy once per
selected folder, and writes status, Unicode logs, and the final result.


15. Important Robocopy options
------------------------------

  /E          include subfolders and empty folders
  /XJ         do not follow junctions
  /FFT        use two-second timestamp tolerance
  /XO         skip source files older than the destination
  /MT:8       use eight copy threads by default
  /R:1 /W:3   limit retries and retry delay
  /COPY:DAT   copy data, attributes, and timestamps
  /DCOPY:DAT  preserve directory data, attributes, and timestamps
  /UNILOG+    append Unicode log output

/MIR and /PURGE are intentionally never used.


16. Limitations
---------------

  - No Volume Shadow Copy is created; locked files may be skipped.
  - This is not a complete Windows system image.
  - Applications and AppData are not fully backed up.
  - Historical file versioning is not provided.
  - Library folders redirected to network shares (paths without a drive
    letter) are not supported.
  - Hardware damage can still cause read or write errors.


17. Exit codes
--------------

  0     Operation completed successfully
  1     Interactive operation declined before starting
  10    Validation, preflight, approval, or general script error
  20    Cooperative cancellation or closed GUI
  >=8   Possible Robocopy copy error when passed through
